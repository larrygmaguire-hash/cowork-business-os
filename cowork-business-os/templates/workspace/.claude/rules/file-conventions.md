## Naming Conventions

### Folders

- **Operational folders:** Title Case with spaces (`Projects`, `Documentation`, `Infrastructure`)
- **Project folders:** Project ID + space + Title Case name (e.g., `P001 Website Redesign`). The ID is generated by `/newproject` using the format configured in `.claude/config/integrations.json` (`projectNumbering` section). Default: `P` prefix, no separator, 3 digits.
- **Client folders:** Title Case with hyphens (`Client-Name`)
- **Archive folders:** Same name as original project folder (including project ID)
- **Department folders:** Numeric prefix + Title Case with spaces. Departments are numbered `01`–`07` (e.g., `01 Finance`, `02 Human Resources`, `03 Sales`). Subfolders use the parent's number with a decimal suffix (e.g., `01.1 Accounts Payable`, `01.2 Accounts Receivable`). The full department numbering is:

  | # | Department |
  |---|-----------|
  | 01 | Finance |
  | 02 | Human Resources |
  | 03 | Sales |
  | 04 | Marketing |
  | 05 | Operations |
  | 06 | Products |
  | 07 | Legal |

  **Subfolder management — MANDATORY:** Default subfolders are suggestions. When creating a new subfolder inside a department:
  1. List existing subfolders to find the highest decimal number (e.g., `01.5` in `01 Finance/`)
  2. Assign the next sequential number (e.g., `01.6`)
  3. Create the folder with the format `{XX.N} {Name}` in Title Case
  4. Update the department's `CLAUDE.md` subfolder table to include the new entry

  When removing a subfolder, remove its row from the department CLAUDE.md table. Do NOT renumber remaining subfolders — gaps are acceptable.

  Department CLAUDE.md files are client-owned (`neverTouch`) and are never overwritten by engine updates.

### Files

- **Structural files:** UPPERCASE, no date prefix (e.g., `CLAUDE.md`, `README.md`, `TEMPLATE.md`, `PRD.md`)
- **Working files:** Date prefix + Title Case, hyphen-separated: `YYYY-MM-DD-File-Name.ext` (e.g., `2026-03-19-Q2-Content-Calendar.xlsx`, `2026-03-15-Brand-Guidelines-Update.md`, `2026-01-15-Client-Agreement.pdf`). The date is the file creation date.
- **No spaces in filenames** — use hyphens to separate words
- **Date prefix applies to all user-created content files.** It does NOT apply to:
  - Structural files (CLAUDE.md, README.md, PRD.md, CHANGELOG.md, ROADMAP.md)
  - Component files inside `.claude/` (rules, skills, commands, agents)
  - Configuration files (settings.json, integrations.json, state.json, engine-manifest.json)
  - Git/system files (.gitignore, .gitkeep)

### Component Files (inside `.claude/`)

| Component | Pattern | Examples |
|-----------|---------|----------|
| Skill folder | `gerund-form-hyphenated` | `drafting-documents`, `processing-spreadsheets` |
| Skill file | `SKILL.md` | Always `SKILL.md` inside the skill folder |
| Command file | `lowercase.md` | `newproject.md`, `status.md` |
| Agent file | `lowercase-hyphenated.md` | `document-processor.md` |
| Rule file | `lowercase-hyphenated.md` | `file-conventions.md`, `content-defaults.md` |

### General Rules

- Apply conventions to **new files only** — do not rename existing files, as this may break cross-references and scripts
- Keep filenames descriptive but concise
- Avoid special characters other than hyphens

## What Stays on Cloud Storage

Heavy files should NOT be saved to this workspace. Keep on cloud storage:

- PDFs, Word documents, PowerPoints, Excel files
- Images, videos, audio files
- Large datasets (over 1MB)
- CAD files, design assets

**Cloud storage path:** [CLOUD_STORAGE_PATH]

When referencing external files, use consistent path notation:
- `[Drive]/Contracts/2026-01-client-agreement.pdf`
- `[Drive]/Marketing/logo-assets.zip`

## Never Create Files In

- Repository root (use appropriate subfolder)
- `.claude/` (unless instructed by administrator)
