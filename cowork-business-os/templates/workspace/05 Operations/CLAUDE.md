# Operations

> **SETUP REQUIRED:** This file contains default settings. When you first use this folder, Claude will ask you to configure it for your organisation — what you store here, which subfolders you need, and any specific rules or conventions. Update this file with your answers so Claude follows your preferences in future sessions.

## Purpose

Day-to-day business operations — processes, vendor management, logistics, and quality control.

## Subfolders

| Folder | Use For |
|--------|---------|
| `05.1 Processes/` | SOPs, process documentation, workflow maps |
| `05.2 Vendors & Suppliers/` | Supplier agreements, contact lists, performance reviews |
| `05.3 Logistics/` | Delivery schedules, inventory tracking, distribution plans |
| `05.4 Quality/` | Quality standards, inspection records, improvement plans |

## Configuration

**Status:** Unconfigured — using defaults.

Customise the following for your organisation. Delete or update each item once decided:

- **What goes here vs Projects:** Standing operational processes (SOPs, vendor lists) belong here. Operational improvement initiatives (e.g., a new system rollout, a process redesign) may be better suited to `Projects/`.
- **What goes here vs other departments:** Operations overlaps with many functions. The principle is: if it governs how work gets done day-to-day, it belongs here. If it belongs to a specific department's function (e.g., HR policy, marketing campaign), it goes there.
- **Subfolders:** Add, rename, or remove subfolders to match your needs. The defaults above are suggestions. Numbering and table updates are handled automatically — see `.claude/rules/file-conventions.md`.
- **Naming conventions:** See `.claude/rules/file-conventions.md` for file and folder naming rules.
- **Heavy files:** Signed agreements, CAD drawings, and large datasets should remain on cloud storage. Reference them by path.
