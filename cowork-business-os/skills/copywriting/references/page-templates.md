# Page Templates

Page structure templates organised by page type.

## Page Layouts

### Varied Layout (Most Pages)
Alternating section types: text block, feature grid, testimonial, text + image, CTA band. Prevents visual monotony.

### Compact Layout (Pricing, Comparison)
Dense information, tables, feature lists. Minimal decoration. Let the data speak.

### Enterprise Layout (Corporate, Partnership)
Formal structure: problem statement, solution overview, case studies, integration details, security/compliance section, contact form.

### Product Launch Layout
Hero with product visual, feature highlights, demo video/GIF, social proof, pricing, FAQ, launch CTA.

## Page-Specific Guidance

### Homepage
- Hero: one-sentence value proposition + CTA
- Social proof bar (logos, numbers)
- 3 key benefits (icon + heading + 1 sentence each)
- How it works (3 steps)
- Testimonial
- Final CTA

### Landing Page
See `landing-page-framework.md` for StoryBrand SB7 structure.

### Pricing Page
- Clear tier names (avoid clever names — use descriptive: Starter, Professional, Enterprise)
- Feature comparison table
- Highlight recommended tier
- FAQ addressing pricing objections
- Annual vs monthly toggle if applicable
- "Not sure?" section with comparison guide or contact option

### Feature Page
- Hero: feature name + one-line benefit
- Problem this feature solves
- How it works (screenshot or diagram + explanation)
- Specific use case or example
- Related features
- CTA: try it / see it in action

### About Page
- Who you are (2-3 sentences, not a novel)
- What you believe (values, stated as positions, not platitudes)
- Team section (if applicable — names, roles, photos)
- Timeline or milestones (optional)
- CTA: work with us / contact / careers

### Product Page
- Product name and one-line description
- Key benefits (3-5)
- Feature details with visuals
- Social proof (reviews, ratings, testimonials)
- Pricing
- FAQ
- CTA: buy / add to cart / request demo

## Section Writing Tips

### Problem Section
- Name the problem in the customer's words
- Be specific (not "businesses struggle with efficiency")
- Quantify the cost where possible ("costs the average team 8 hours per week")

### Benefits Section
- Lead with the outcome, not the feature
- "See every client interaction in one view" not "Unified client database"
- One benefit per block. Do not combine.

### How It Works Section
- 3-4 steps maximum
- Number them
- Each step: one verb, one outcome
- Visual per step if possible

### Testimonial Section
- Real names and roles
- Specific results ("Saved us 12 hours a week") over vague praise ("Great product")
- Photos increase credibility
- Mix of audience types if available

### FAQ Section
- Address genuine objections, not softball questions
- Each answer: 2-3 sentences, ending on a positive
- Common questions: pricing, implementation time, support, integrations, security
