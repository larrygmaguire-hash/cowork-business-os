## Standard project folder structure

A typical project folder contains:

- `CLAUDE.md` -- this file. Project-specific context, scope, deliverables, conventions.
- `Research/` -- background reading, references, source materials
- `Drafts/` -- work in progress (documents, spreadsheets, presentations)
- `Final/` -- completed deliverables ready for delivery or publication

The exact subfolders vary by project type. Adjust to fit the work.

## What to put in this CLAUDE.md when you create a real project

When creating a new project (via `/creating-projects`), the skill generates a CLAUDE.md with:

- **Scope:** what the project covers and what it does not
- **Deliverables:** the specific outputs the project will produce
- **Stakeholders:** who is involved, who approves, who consumes the output
- **Constraints:** deadlines, budgets, approval requirements
- **Current state:** where the work stands right now
- **Conventions:** any project-specific naming, formatting, or process rules that override the workspace defaults

The full operational context (autonomy levels, file conventions, state management) is inherited from `.claude/CLAUDE.md`. You do not need to repeat those rules here.
