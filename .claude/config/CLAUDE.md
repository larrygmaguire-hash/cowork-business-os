# Config

Integration configuration for the workspace. Holds non-secret settings that skills reference: API endpoints, service IDs, cloud storage paths, calendar IDs, and similar.

## What goes here

- `integrations.json` -- structured configuration file for integrations (Google Calendar IDs, cloud storage paths, third-party API endpoints, project numbering format)

## What does NOT go here

- API keys, tokens, passwords, or any credentials -- these go in environment variables or a `.env` file (already gitignored). Never commit secrets.
- User-specific paths (for example, your local Dropbox path) if the workspace is shared with others
- Anything generated by skills at runtime (those go in `.claude/state/` or appropriate workspace folders)

## Structure of integrations.json

The file uses a flat or nested JSON structure. Common sections include:

```json
{
  "projectNumbering": {
    "prefix": "P",
    "separator": "",
    "digits": 3
  },
  "cloudStorage": {
    "primary": "[Path or label, e.g. 'Google Drive']",
    "rootPath": "[Optional: absolute path on this machine]"
  },
  "calendar": {
    "selectedCalendars": [],
    "provider": ""
  },
  "thirdParty": {
    "exampleService": {
      "endpoint": "",
      "accountId": ""
    }
  }
}
```

The `/setup` skill creates a default `integrations.json` based on your answers. Edit it directly to add new integrations.

## Skills that read this file

- `creating-projects` -- reads `projectNumbering` to format new project IDs
- `session-briefing` -- reads `calendar.selectedCalendars` if calendar integration is enabled
- Any custom skill that needs cross-cutting configuration

If a skill needs a setting that does not yet exist, the skill should:
1. Check if the setting exists
2. If not, ask the user and write the setting back to integrations.json
3. Use the setting on subsequent invocations
